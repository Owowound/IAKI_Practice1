# Алгоритм создания сетки

Для создания сетки производится несколько действий:
1. На этапе размещения актора создается UBoxComponent, который растягивается вместе с актором для полного покрытия лабиринта.
2. В соответствии с указанным размером ячейки вычисляется TopLeft(координаты данной точки зависят от перспективы, названа так для удобства), которая определяет локацию одной из угловых ячеек.
3. С помощью двойного цикла по ячейкам создаются и настраиваются ячейки(для проверки заблокированности используется отдельный UBoxComponent для проверки коллизий и тэгов **Walls**)

В итоге получаем готовый вектор из векторов с ячейками CellData, которые определяют основную информацию по ячейкам(координаты, активность, родительские связи и т.д.).

# Определение ячеек Start/End

Для определения необходимо задать актор Start/End в класс на поле.

В самом начале с помощью отдельного метода координаты преобразуются в индекс ячейки с помощью округления. 

*Замечание: ячейкам ожет определяться, если она назодится за пределами лабиринта, но в том случае, если активная ячека лабиринта находится в прямой видимости по перпендикуляру от Start/End*

# Алгоритмы 

## Дейкстра

Для алгоритма используется BFS от точки Start. 

Для каждой точки в очереди проверяются все непосещенные соседи (4 направления) и добавляются в очередь.  Как только найдена точка End цикл BFS завершается и начинается восстановление пути от End с помощью проверки родителя данной точки, вычисленного на этапе BFS.

В процессе все точки добавляются в отдельные структуры для последующих визуализаций: визуализации процесса и восстановления пути.

## A*

Для алгоритма используется алгоритм подобный BFS, но с приоритетной очередью. Очередь сортирует по убываюнию особого расстояния, которое представляет собой FCost = distance + heuristic, где distance- это расстояние, пройденное со старта, а heuristic- это расстояние до конца, игнорирующее препятствия.

В остальном сам алгоритм и восстановление путей аналогичны.

# Запуск

Для сравнения используются 2 идентичных алгоритма и 2 объекта APathfinder. Для изменения алгоритма следуется менять флаг `IsDijkstra` в параметрах объекта на сцене. При запуске нижний объект использует Дейкстру, а верхний - A*.

При запуске рекомендую устанавливать размер ячеек 20, так как при построении сетки с другими размерами(к примеру 15) она может быть построена некорректно из-за реализации сетки(к примеру в узком проходе размером около 20 при размере 15 не будет построена ячека из-за стыка ячеек посередине и онаружения коллизий). При этом слишком малые размеры будут влиять на производительность из-за большого числа дебаг-объектов на сцене.
